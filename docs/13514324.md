# 新的打包利器 Rolldown

尤大团队在 [2026年3月12日](https://cn.vite.dev/blog/announcing-vite8#migrating-to-vite-8)，发布了新一代的打包工具，[Rolldown](https://rolldown.rs/guide/getting-started)，结束了esBuild+Rollup双环境打包的架构体系。

下面我们就简单说一说`Rolldown`为我们带来了什么。

## 核心原理

`Rolldown` 本质上是一个用 `Rust` 编写的 `JavaScript` 打包工具，其设计目标是成为 `Vite` 在生产环境构建时的底层打包器，替代目前使用的 `Rollup`。

它的核心原理可以概括为：

1. **架构兼容Rollup**：`Rolldown` 提供了与 `Rollup` **几乎完全兼容的插件接口和配置选项**。这意味着 `Rollup` 生态中大量的插件可以直接在 `Rolldown` 中使用，降低了迁移成本。其 `API`（`rolldown()`、`build()`、`watch()`）也与 `Rollup` 保持高度一致。
2. **Rust 核心与 Node.js绑定**：所有关键的打包逻辑（如模块解析、依赖图构建、代码转换、摇树优化、代码生成）都用 `Rust` 实现。通过 `N-API` 为 `Node.js` 提供原生绑定，使得 `JavaScript` 层面可以高效地调用底层的 `Rust` 功能。
3. **并行与并发优先**：`Rust` 语言天生对并发编程有很好的支持。`Rolldown` 在设计之初就注重利用多核 `CPU` 的能力，在模块解析、转换和代码生成等阶段**尽可能并行处理**，这是它实现高性能的基础。
4. **支持多种模块格式**：能够处理 `ESM`、`CommonJS`、`IIFE` 等多种模块格式，并能在输出时进行格式转换。

## 为什么 `Rolldown` 快？

`Rolldown` 的性能优势主要来源于**语言特性、架构设计和工程优化**的叠加，具体体现在以下几个方面：

| 维度 | 传统 `JS` 打包器（`Rollup`、`Webpack`） | `Rolldown`（`Rust`） | 速度优势来源 |
| :--- | :--- | :--- | :--- |
| **语言层** | `JavaScript`/`TypeScript`，运行在 `Node.js`（解释型或 `JIT` 编译）。 | `Rust`，编译为**原生机器码**，直接运行在 `CPU` 上。 | 消除 `JIT` 编译开销，启动更快，计算密集型任务（如 `AST` 解析）性能显著提升。 |
| **并行处理** | 受限于 `Node.js` 的单线程事件循环，虽然 `Worker threads` 可实现并行，但代价和复杂度较高。 | 利用 `Rust` 的**多线程**和**无锁数据结构**，在模块解析、转换等阶段实现细粒度的并行，充分利用多核 `CPU`。 | 现代服务器多核 `CPU` 利用率更高，整体吞吐量大幅提升。 |
| **内存管理** | 依赖 `Node.js` 的垃圾回收（`GC`），在大型项目打包时 `GC` 可能成为性能瓶颈。 | 采用 `Rust` 的**所有权系统**，在编译期确定内存生命周期，运行时无 `GC` 开销。 | 内存使用更高效、稳定，避免了因 `GC` 暂停导致的性能波动。 |
| `AST` **与代码生成** | 使用 `JS` 实现的解析器（如 `acorn`、`swc`）或依赖第三方原生库。 | 使用 `Rust` 编写的解析器（部分基于 `oxc` 项目），并且整个 `AST` 到代码生成的流程都在原生层完成。 | 减少 `JS` ↔ 原生层之间的反复调用，整体流水线效率更高。 |
| **冷启动与缓存** | 项目初次启动时需要加载大量 `JS` 代码并进行解析。 | 原生二进制启动极快，且能更高效地利用文件系统缓存。 | 尤其适合 `CLI` 工具、`vite build` 等需要频繁启动的场景。 |

## 总结

`Rolldown` 不是简单地用 `Rust` 重写 `Rollup`，而是一个在保持上层 `API` 兼容性的同时、**底层完全重新设计的高性能打包器**。

它通过 `Rust` 的 **原生性能**、**精细的并行控制**与**零开销的内存管理**，解决了以 `Rollup` 为代表的传统 `JS` 打包器在处理大型项目时的性能瓶颈。

目前它已进入 `RC` 阶段，并且是 `Vite` 未来**生产构建的默认打包器**，其性能优势将直接惠及大量 `Vite` 用户。

如果你想深入了解其技术实现细节，可以查阅 [官方文档](https://rolldown.rs/guide/getting-started) 中更深入的部分。
