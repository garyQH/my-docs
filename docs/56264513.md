## 字符串扫描器与模板替换

如果不使用正则表达式，最经典且最贴近 `Vue` 源码底层逻辑的方法是 **「字符串扫描器」**（`String Scanner`）。

这种方法通过遍历字符串的每一个字符，利用 **状态标记** 来识别何时进入了花括号、何时离开了花括号。

这种方式虽然代码量稍多，但 **性能更好**，且更容易扩展（例如处理嵌套或特殊转义）。

> 单次循环扫描把时间复杂度稳定在 `O(n)`，并避免复杂正则在大文本上的回溯风险。

---

### 基于状态扫描的实现方案

可以维护一个 `stack`（栈）或简单的计数器来追踪花括号。下面示例用指针与 `indexOf` 消费 `{{` … `}}` 区间：

```js
function nonRegexRender(template, data) {
  nuxt markdown let result = '';
  let i = 0;

  while (i < template.length) {
    // 检查是否遇到了 '{{'
    if (template[i] === '{' && template[i + 1] === '{') {
      let startIndex = i + 2; // 跳过 '{{'
      let endIndex = template.indexOf('}}', startIndex);

      if (endIndex === -1) {
        // 如果没找到对应的 '}}'，说明格式不对，直接跳出
        result += template.slice(i);
        break;
      }

      // 提取变量名并去空格
      const rawKey = template.slice(startIndex, endIndex);
      const key = rawKey.trim();

      // 替换数据（这里只演示了单层对象，支持深度访问可复用上个回答的方法）
      result += data[key] !== undefined ? data[key] : `{{${rawKey}}}`;

      // 将指针移动到 '}}' 之后
      i = endIndex + 2;
    } else {
      // 普通文本直接拼接
      result += template[i];
      i++;
    }
  }

  return result;
}

// 测试
const tpl = "你好 {{ name }}，欢迎来到 {{ city }}！";
const info = { name: "阿强", city: "大连" };

console.log(nonRegexRender(tpl, info));
// 输出: "你好 阿强，欢迎来到 大连！"
```

---

### 这种方法的优势

1. **更强的可控性**：正则在处理复杂边界（例如 `{{ name + '}}' }}` 这类字符串与定界符纠缠）时会变得极难维护，而 **手动扫描** 可以通过增加分支逐步覆盖。
2. **`Vue` 编译器的思路**：`Vue` 的 `compiler-core` 在解析模板（`Template Parser`）时，本质上也是维护指针，通过 `advanceBy(n)` 一类 API 消费字符。
3. **避免回溯风险**：在大文本下，过于复杂的正则可能带来 **灾难性回溯**；而 **线性扫描** 的行为更可预期。

---

### 两种方式对比

| 特性 | 正则表达式（`replace`） | 字符扫描（`indexOf` / `while`） |
| :--- | :--- | :--- |
| **代码量** | 极简，通常 1～3 行 | 较多，需处理指针移动 |
| **易读性** | 高（若熟悉正则） | 中等（逻辑更直白） |
| **灵活性** | 较低，难处理复杂嵌套 | 极高，可叠加语法规则 |
| **性能** | 现代引擎优化得很好 | 大规模文本下往往更稳健 |

---

若希望对照真实实现，可参考 `Vue` 官方仓库中模板解析与 `compiler-core` 的设计说明：[Vue.js 文档 · 渲染机制](https://cn.vuejs.org/guide/extras/rendering-mechanism.html) 与源码中的 `compiler-core` 解析流程，便于理解从字符到 **`Tokens`** 再到 **`AST`**（抽象语法树）的链路。
