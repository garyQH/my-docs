## 异步遍历

在 `JavaScript` 中处理一组异步任务的遍历，最关键的在于你希望它们是**串行（一个接一个）执行，还是并行（同时开始）**执行。


### 1. 并行执行（效率最高）

如果你不关心任务执行的顺序，只希望它们尽快完成，这是最推荐的做法。通过 `Array.prototype.map` 将数据转为 `Promise` 数组，再用 `Promise.all` 统一等待。


```js
const ids = [1, 2, 3];

async function fetchData() {
  // map 不会等待 await，它会立即返回一个包含多个 Promise 的数组
  const promises = ids.map(async (id) => {
    const res = await fetch(`https://api.example.com/data/${id}`);
    return res.json();
  });

  // 并行启动所有请求，直到全部完成
  const results = await Promise.all(promises);
  console.log('全部完成:', results);
}
```
- 优点： 速度快，总耗时约等于最慢的那个任务。
> 注意： 如果其中一个任务报错，整个 `Promise.all` 会立即抛出异常。如果不希望被单个错误阻断，可以使用 `Promise.allSettled`。

### 2. 串行执行（一个接一个）

如果你有严格的顺序要求（比如第二个请求依赖第一个的结果），或者需要控制频率避免服务器压力过大，应使用 `for...of` 循环。


```js
async function fetchInOrder() {
  const results = [];
  
  for (const id of ids) {
    // 循环会在这里暂停，直到当前请求完成
    const res = await fetch(`https://api.example.com/data/${id}`);
    const data = await res.json();
    results.push(data);
    console.log(`已处理 ID: ${id}`);
  }
  
  return results;
}
```

- 优点： 逻辑简单，顺序可控。
- 缺点： 速度慢，总耗时等于所有任务耗时的总和。

### 3. 控制并发数

如果你有 **1000** 个任务，`Promise.all` 会瞬间发出 **1000** 个请求，这可能导致浏览器卡死或被服务器封 `IP`。此时需要手写一个简易的并发控制，或者使用 `p-limit` 等库。

```js
async function limitConcurrency(tasks, limit) {
  const results = [];      // 存储所有任务的 Promise，用于最后获取结果
  const executing = new Set(); // 【关键】正在执行中的任务池

  for (const task of tasks) {
    // 1. 开始执行当前任务
    // Promise.resolve().then(...) 确保 task 是以异步 Promise 形式启动
    const p = Promise.resolve().then(() => task());
    
    // 2. 将这个 Promise 放入结果集和执行池
    results.push(p);
    executing.add(p);
    
    // 3. 【重点】任务完成后，自动从执行池中把自己踢出去
    // 不管成功还是失败（finally），只要结束了，池子就空出一个位置
    p.finally(() => executing.delete(p));

    // 4. 【核心控制】如果执行池里的任务数量达到了限制（limit）
    if (executing.size >= limit) {
      // 这里的 await 会阻塞 for 循环
      // Promise.race(executing) 会等待执行池中“最快完成”的那一个任务
      // 只要有一个任务完成了，循环就会继续往下走，处理下一个 task
      await Promise.race(executing);
    }
  }

  // 5. 最后，用 Promise.all 等待所有任务全部收尾
  return Promise.all(results);
}
```

| 场景      | 推荐写法 |
| ----------- | ----------- |
| 追求速度，无序      | `Promise.all` + `map`       |
| 严格顺序，逻辑简单   | `for...of` 循环        |
| 需要处理部分报错   | `Promise.allSettled`       |
| 海量任务（如爬虫）   | 手写并发控制或使用第三方库      |
