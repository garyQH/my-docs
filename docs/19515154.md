# Vue2/3的对比

`Vue 3` 的底层实现确实从 `Object.defineProperty` 切换到了 `Proxy`，但其**核心设计哲学（依赖收集 + 发布订阅模式）是一脉相承的**。

你可以把 `Vue 2` 到 `Vue 3` 的演进理解为：**"换了更强的引擎，但赛车的逻辑没变"。**

---

## 1. 核心模型对比

| 角色 | `Vue 2` (核心：`defineProperty`) | `Vue 3` (核心：`Proxy`) | 变化本质 |
| :--- | :--- | :--- | :--- |
| **监听器 (`Observer`)** | `Observer`：递归遍历对象属性，将其转为 `getter`/`setter`。 | `Reactive` / `Ref`：使用 `Proxy` 代理整个对象。 | 从"属性拦截"变为"对象代理"。 |
| **依赖收集池 (`Dep`)** | `Dep`：每个属性都有一个专属的 `Dep` 实例，存放在闭包里。 | `targetMap`：使用 `WeakMap` 统一存储对象、属性与副作用的映射。 | 从"分散存储"变为"全局映射管理"。 |
| **订阅者 (`Watcher`)** | `Watcher`：分为渲染、计算、侦听三种 `Watcher`。 | `Effect`（副作用）：更底层的函数封装，组件渲染本质就是一个 `Effect`。 | 从"面向类(`Class`)"变为"面向函数(`Function`)"。 |

---

## 2. 深度讲解：`Vue 3` 的变化

### ① 从 `Dep` 到 `targetMap` (全局依赖追踪)

`Vue 2` 的依赖存在属性的闭包里，而 `Vue 3` 使用了一个全局的、结构化的 **`WeakMap`** 来存储依赖关系，结构如下：

- `WeakMap` (`targetMap`)：键是 `原始对象`，值是 `depsMap`。
- `Map` (`depsMap`)：键是 `属性名`，值是 `depSet`。
- `Set` (`depSet`)：存储所有依赖于该属性的 `Effect` 函数。

> 为什么用 `WeakMap`？因为它是弱引用，当原始对象被销毁时，对应的依赖关系也会被垃圾回收，防止内存泄漏。

### ② 从 `Watcher` 到 `Effect` (副作用)

`Vue 3` 把所有响应式逻辑归纳为 **`Effect`（副作用）**。

- 当一个函数运行时访问了响应式数据，`Vue` 就会把当前正在运行的函数（`activeEffect`）存入对应的 `depSet` 中。
- 当数据变化时，直接遍历这个 `Set`，执行里面所有的函数。

### ③ 从 `defineProperty` 到 `Proxy` (性能杀手锏)

**为什么 `Proxy` 更好？**

1. **性能**：不需要在初始化时递归遍历所有属性。只有在访问到深层属性时，`Vue 3` 才会动态地对那一层进行代理（`Lazy Proxing`）。
2. **功能**：`Vue 2` 监听不到数组下标修改、属性的新增和删除；`Vue 3` 的 `Proxy` 拦截的是整个对象，这些统统没问题。

---

## 3. `Vue 3` 响应式的"工作流"

1. **初始化**：调用 `reactive()`，返回一个 `Proxy` 对象。
2. 依赖收集 (`Track`)：组件渲染，触发 `Proxy` 的 `get`。`Vue` 在全局 `targetMap` 中找到该属性，并把渲染 `Effect`存入其 `depSet`。
3. 数据修改 (`Trigger`)：修改属性，触发 `Proxy` 的 `set`。
4. **分发更新**：`Vue` 从 `targetMap` 找到对应的 `depSet`，取出所有 `Effect` 并执行。
5. **批量更新**：所有的 `Effect` 并不是立即执行，而是放入微任务队列中进行**批处理**，最后完成 `DOM` 更新。

---

## 4. 总结对比：`Vue 2` vs `Vue 3`

| 维度 | `Vue 2` | `Vue 3` |
| :--- | :--- | :--- |
| **初始化开销** | 大（需一次性递归所有属性） | 小（只代理外层，深层按需代理） |
| **动态增删属性** | 需 `$set` / `$delete` | 直接操作，原生支持 |
| **数组支持** | 需重写数组原型方法 | 原生支持 |
| **代码组织** | `Option API`（逻辑分散） | `Composition API`（逻辑聚合） |
| **底层代码量** | 较重（`Class` 实现） | 轻量（`Function` 配合闭包实现） |
