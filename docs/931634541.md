## LLM、Agent、MCP 与 Skills

在 `AI` 领域，**MCP**（`Model Context Protocol`）是当前提升大模型 **Agent**（智能体）能力的一个核心技术协议。

为了理解它们之间的关系，我们可以把大模型想象成一个**超级大脑**，而 `MCP` 就是它的**通用插座**。

---

### 1. 什么是 `Agent`？

如果说普通的大模型（如 `GPT`）是一个**聊天机器人**，那么 `Agent` 就是一个**行动派**。

- **核心特征：** 它不仅能说话，还能自主决策、规划步骤并使用工具来完成复杂任务。
- **公式：** Agent = 大模型 (LLM) + 规划 (Planning) + 记忆 (Memory) + 工具使用 (Tool Use)。

### 2. 什么是 `MCP`？

**MCP** 是由 `Anthropic` 公司推出的**模型上下文协议**。

它的出现是为了解决一个痛点：每个 `AI` 应用连接外部数据（如 `GitHub`、`Google Drive`、本地数据库）的方式都不一样，开发者需要重复造轮子。

- **本质：** 它是一套开放标准，让 `AI` 模型能够以**统一的方式**访问各种外部数据源和工具。
- **类比：** 就像 `USB` 接口出现之前，鼠标、键盘、打印机都有各自的接口；有了 `USB`（`MCP`），任何设备只要符合标准，插上就能用。

### 3. 什么是 `Skill`？

在 `Agent` 的语境下，**Skill** 是 `Agent` 能够执行的具体能力。通过 `MCP` 协议，开发者可以给 `Agent` 赋予各种**技能包**。

- **Skill 的构成：** 通常包含一段代码逻辑和一份 `JSON Schema` 描述（告诉大模型这个技能是干什么的、需要什么参数）。
- **实际例子：**
  - **搜索技能：** 能够调用 `Google Search` 获取实时信息。
  - **代码技能：** 能够在本地终端执行 `Python` 脚本或操作 `Git` 仓库。
  - **数据库技能：** 能够直接查询 `SQL` 数据库并总结报表。

---

### 三者的协同工作流程

我们可以用**面试一个新员工**来串联这三个概念：

| 概念 | 角色 | 作用 |
| :--- | :--- | :--- |
| **Agent** | **新员工** | 负责理解老板的任务（`Prompt`），并思考怎么做。 |
| **Skill** | **员工的技能证书** | 比如**精通 Python**、**会用 Excel**，决定了员工能干什么。 |
| **MCP** | **员工的工卡/权限系统** | 员工通过这张工卡，可以合法地进入公司的机房、调取数据库。 |

**一个典型的场景：**

1. **用户：**

   > 帮我分析一下本地 `src` 目录下的代码逻辑，并写一份文档。
2. **Agent：** 意识到自己需要读取文件。
3. **MCP：** 提供了一个标准的**文件系统连接器**。
4. **Skill：** `Agent` 调用了 `list_files` 和 `read_file` 这两个技能。
5. **结果：** `Agent` 获取内容，分析后给出回复。

---

### 为什么这很重要？

在 `MCP` 出现之前，如果你想让 `AI` 连接你的 `Notion`，你需要写很多适配代码。

现在，只要社区有一个 **MCP Server** 实现了 `Notion` 接口，任何支持 `MCP` 的 `Agent` 框架（如 `Claude Desktop`、`Goose` 等）都能 **瞬间获得** 操作 `Notion` 的技能。

这大大降低了构建 **全能 Agent** 的门槛。
