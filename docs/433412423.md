## Paint Worklet

**Paint Worklet**（也叫 **CSS Custom Paint**）是 `CSS` `Houdini` 系列 `API` 中最成熟的一个。

它的核心价值在于：**让开发者可以用 `Canvas` 的绘画能力，直接编写 `CSS` 属性（如 `background-image` 或 `border-image`）。**

在大厂（如 `Google`、字节跳动、或者一些高端设计驱动的 `SaaS` 公司）中，它主要用于解决 **极端的性能优化** 和 **动态视觉特效**。

---

## 一、 大厂的实用案例

### 1. 动态自适应的背景（如气泡、波浪、粒子）

- **传统方案**：使用巨大的 `GIF` 或视频作为背景，或者在主线程用 `Canvas` 实时画。前者费流量，后者占 `CPU` 导致页面滚动卡顿。
- **Paint Worklet 方案**：编写一个 `Paint` 脚本处理粒子逻辑。因为它运行在浏览器的**渲染阶段**（`Rendering Pipeline`），且拥有独立的 `Worklet` 线程，它的渲染性能接近原生，且不会阻塞 `JS` 业务逻辑。

### 2. 完美的**平滑圆角**（`Apple Style Squircle`）

- **传统方案**：`border-radius` 只能画圆弧，无法实现 `iOS` 那种完全平滑的超椭圆圆角。
- **Paint Worklet 方案**：定义一个 `smooth-corners` 属性，直接通过数学公式画出完美的圆角。像 `Figma` 网页版或对设计要求极高的品牌官网（如 `Apple` 风格页面）会采用这种技术。

### 3. 实时占位图（`Skeleton Screen`）

- **实用场景**：在大数据量列表加载时，大厂会用 `Paint Worklet` 生成带有动态闪烁效果的骨架屏，而不是加载一张占位图。这能减少 `HTTP` 请求，且动画极其丝滑。

### 4. 复杂的边框与分割线

- 在某些复杂的电商大促页面（如双11），背景可能有非常复杂的切角、渐变和纹理叠加。用 `CSS` 组合极其痛苦，用图片又不够灵活。`Paint Worklet` 可以根据元素的 `width` 和 `height` 动态绘制这些复杂的图形。

---

## 二、 如何使用（核心 3 步）

### 第一步：编写 `Worklet` 脚本（`my-paint.js`）

就像写 `Canvas` 一样，定义一个类并注册它。

```js
// my-paint.js
class SmoothCorners {
    // 允许接收 CSS 变量
    static get inputProperties() { return ['--smooth-amount']; }

    paint(ctx, geom, properties) {
        const amount = properties.get('--smooth-amount').toString() || 0.6;
        const w = geom.width;
        const h = geom.height;

        ctx.fillStyle = 'black';
        // 使用 Canvas API 绘图
        ctx.beginPath();
        // 这里简化了，实际会是超椭圆数学公式
        ctx.arc(w/2, h/2, Math.min(w,h)/2, 0, Math.PI * 2);
        ctx.fill();
    }
}

registerPaint('smooth-corners', SmoothCorners);
```

### 第二步：在 `HTML` 中引入

```html
<script>
    if ('paintWorklet' in CSS) {
        CSS.paintWorklet.addModule('my-paint.js');
    }
</script>
```

### 第三步：在 `CSS` 中调用

就像使用普通函数一样，使用 `paint(注册名)`。

```css
.box {
    width: 200px;
    height: 200px;
    /* 调用自定义绘制逻辑 */
    background-image: paint(smooth-corners);
    /* 传递参数给 Worklet */
    --smooth-amount: 0.8;
}
```

## 三、 `Paint Worklet` 的优势与局限

### 优势

- **性能极高**：它在渲染流水线中运行，绕过了昂贵的 `DOM` 操作。
- **响应式**：图形随元素大小自动重绘，不需要手动监听 `resize` 事件。
- **解耦**：视觉逻辑（`JS`）与样式定义（`CSS`）完美分离。

### 局限（大厂也会头疼的地方）

- **兼容性**：目前主要是 **`Chromium` 引擎（`Chrome`, `Edge`, 浏览器内核）** 支持良好。`Safari` 和 `Firefox` 进度缓慢。

<div class="Alert">

大厂策略：通常将其作为**渐进增强**，支持的浏览器显示炫酷效果，不支持的显示普通 `border-radius`。

</div>

- **无 `DOM` 访问权**：你不能在里面操作 `HTML`，只能画图。
- **安全限制**：必须在 `HTTPS` 环境下运行（本地 `localhost` 除外）。

---

## 总结

如果你在做**极简主义、高性能、或者是拥有大量动态视觉元素**的前端项目（比如**高科技、极简风格**的 `Web` 界面），`Paint Worklet` 可以用来生成那些**磨砂玻璃渐变、动态几何背景**，效果会比传统的图片或 `Canvas` 模拟高级得多。
