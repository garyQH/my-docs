## Web Components

`Web Components` 是前端领域一项非常重要但常常被忽视的技术，因为它是**浏览器原生**支持的组件化方案，不依赖于 `React`、`Vue` 或 `Angular` 这样的第三方框架。

随着 [微前端](./119134134.md) 的兴起，`Web Components` 的地位正变得越来越重要。

---

### 一、什么是 `Web Components`？

`Web Components` 是一套不同的技术，允许你创建**可复用**的**定制元素**（它们的机制与原生 `HTML` 元素如 `<button>`、`<input>` 一样），并且可以将它们的功能封装在代码之外，主文档不会受到其内部实现的影响。

**用通俗的话说**：它允许你自定义一个新的 `HTML` 标签，比如 `<user-card>`，这个标签自带样式（`CSS`）和行为（`JS`），你可以像使用 `<video>` 标签一样在任何 `HTML` 页面中使用它，而不用担心它的样式会污染外部页面，或者外部样式会破坏它。

---

### 二、核心组成部分

`Web Components` 的强大源于三项核心技术的结合：

#### 1. `Custom Elements`（自定义元素）

这是一组 `JavaScript` `API`，允许你定义一种新的 `HTML` 标签及其行为。

- **核心：** `window.customElements.define()` 方法。
- **细节：** 你的自定义元素必须继承自 `HTMLElement` 类，并且标签名**必须包含连字符（-）**，例如 `<my-element>`，以区分原生标签。

#### 2. `Shadow DOM`（影子 DOM）

这是实现隔离的关键。它允许你将一个隐藏的、独立的 `DOM` 树附加到一个元素上。

- **核心：** `element.attachShadow({ mode: 'open' })`。
- **细节：** `Shadow DOM` 内部的 `CSS` 样式是**作用域隔离**的。这意味着在 `Shadow DOM` 内部定义的 `.button { color: red; }` 不会影响外部页面的 `.button`，反之亦然。这完美解决了**样式污染**问题。

#### 3. `HTML Templates`（`HTML` 模板）

`<template>` 和 `<slot>` 标签。它们允许你编写不在页面加载时渲染的 `HTML` 结构，但可以在运行时通过 `JS` 实例化。

- **核心：** `<template>` 标签用于定义结构，`<slot>`（插槽）用于占位，允许外部在使用时传入自定义内容。

---

### 三、应用场景

1. **构建通用 UI 组件库**：创建不依赖任何框架（`Vue`/`React`）的组件（如按钮、弹窗、导航栏），供整个大公司的不同技术栈项目共同使用。
2. **微前端隔离**：利用 `Shadow DOM` 的硬隔离特性，作为微前端子应用的容器，彻底阻断 `CSS` 冲突。
3. **遗留系统增量升级**：在一个老旧的 `jQuery` 项目中，直接引入一个现代的 `Web Component` 组件，而无需重构整个项目。
4. **内容分发**：例如，社交媒体（`Twitter`/`Facebook`）提供的 **嵌入帖子** 小部件，通常就是利用类似技术实现的封装。

---

### 四、完整实战：`<user-card>`

我们将创建一个自定义的 `<user-card>`（用户卡片）组件。它有自己的头像、名字，并支持外部传入详细描述（插槽）。

#### 1. `HTML` 定义结构（`index.html`）

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Web Components Demo</title>
    <style>
        /* 外部页面的样式，用来验证隔离性 */
        body { font-family: sans-serif; background-color: #f0f2f5; }
        p { color: gray; } /* 这个 gray 不会影响自定义组件内部的 p */
    </style>
</head>
<body>

    <h1>Web Components 示例</h1>
    <p>我是外部页面的普通段落，我是灰色的。</p>

    <user-card 
        name="张三" 
        avatar="https://randomuser.me/api/portraits/men/1.jpg">
        <span slot="description">高级前端架构师，专注于高性能 Web 应用。</span>
    </user-card>

    <br>

    <user-card 
        name="李四" 
        avatar="https://randomuser.me/api/portraits/women/2.jpg">
        <span slot="description">UI/UX 设计师，热爱极简主义。</span>
    </user-card>

    <script src="userCard.js"></script>
</body>
</html>
```

#### 2. `JavaScript` 实现逻辑（`userCard.js`）

```js
// 1. 定义 HTML 模板结构和样式
const template = document.createElement('template');
template.innerHTML = `
  <style>
    /* 组件内部的专属样式 (Shadow DOM Scope) */
    .user-card {
      display: flex;
      background-color: #fff;
      width: 400px;
      border-radius: 8px;
      box-shadow: 0 4px 8px rgba(0,0,0,0.1);
      padding: 20px;
      margin-bottom: 15px;
      overflow: hidden;
    }
    .avatar {
      width: 80px;
      height: 80px;
      border-radius: 50%;
      margin-right: 20px;
      border: 3px solid #eee;
    }
    .info h3 {
      margin: 0 0 10px 0;
      color: #333;
    }
    /* 这里的 p 是黑色的，不受外部 gray 样式影响 */
    .info p {
      margin: 0;
      color: #666; 
      font-size: 14px;
    }
    .description-slot {
        margin-top: 10px;
        font-style: italic;
        color: #888;
    }
    button {
        margin-top: 10px;
        padding: 5px 10px;
        background-color: #007bff;
        color: white;
        border: none;
        border-radius: 4px;
        cursor: pointer;
    }
  </style>

  <div class="user-card">
    <img class="avatar" src="" alt="user avatar" />
    <div class="info">
      <h3></h3>
      <p>ID: <span class="user-id"></span></p>
      <div class="description-slot">
        <slot name="description">这里是默认描述（如果外部没传）</slot>
      </div>
      <button id="toggle-info">打个招呼</button>
    </div>
  </div>
`;

// 2. 创建一个类继承自 HTMLElement
class UserCard extends HTMLElement {
  constructor() {
    super();

    // 3. 将 Shadow DOM 附加到元素上 (mode: open 意味着可以通过 JS 从外部访问 shadowRoot)
    this.attachShadow({ mode: 'open' });

    // 4. 将模板内容克隆并添加到 Shadow DOM 中
    this.shadowRoot.appendChild(template.content.cloneNode(true));
  }

  // 生命周期钩子：当元素被插入到 DOM 时触发
  connectedCallback() {
    // 5. 获取属性值并渲染到 Shadow DOM 中
    const name = this.getAttribute('name') || '未命名';
    const avatar = this.getAttribute('avatar') || 'https://via.placeholder.com/80';
    
    // 生成一个随机 ID
    const randomId = Math.floor(Math.random() * 1000);

    // 访问 Shadow DOM 内部的元素
    this.shadowRoot.querySelector('h3').innerText = name;
    this.shadowRoot.querySelector('.avatar').src = avatar;
    this.shadowRoot.querySelector('.user-id').innerText = randomId;

    // 6. 添加事件监听 (隔离在组件内部)
    this.shadowRoot.querySelector('#toggle-info').addEventListener('click', () => {
        alert(`你好，我是 ${name}！我的 ID 是 ${randomId}`);
    });
  }

  // 生命周期钩子：当元素从 DOM 中移除时触发
  disconnectedCallback() {
      // 在这里清理定时器或移除全局事件监听
      console.log('组件已卸载');
  }
}

// 7. 定义自定义元素
// 标签名必须有连字符
window.customElements.define('user-card', UserCard);
```

### 总结：`Web Components` 的优缺点

| 优点 | 缺点 |
| :--- | :--- |
| **原生支持**：不依赖任何框架，体积极小。 | **开发体验**：原生的 `DOM` 操作 `API` 比较繁琐，不像 `Vue`/`React` 那样可以用声明式数据绑定。 |
| **完美隔离**：`Shadow DOM` 彻底解决 `CSS` 冲突。 | **跨组件通信**：比起现代框架的全局状态管理（`Pinia`/`Redux`），组件间传递复杂数据较为麻烦。 |
| **真正的复用**：一次编写，随处运行（`React` 中可用，`Vue` 中也可用）。 | **生态**：组件库的数量和成熟度不如 `React`/`Vue` 生态。 |

### 扩展

1. 为了解决原生 `API` 繁琐的问题，谷歌推出了 `Lit` 库，它用类似 `React` 的声明式语法封装了 `Web Components`，极大地提升了开发效率，是目前构建 `Web Components` 组件库的首选方案。

```js
import { LitElement, html, css } from 'lit';
import { customElement, property } from 'lit/decorators.js';

@customElement('my-counter') // 装饰器直接注册标签
export class MyCounter extends LitElement {
  // 1. 定义样式 (自动处理 Shadow DOM 隔离)
  static styles = css`p { color: blue; }`;

  // 2. 声明响应式属性
  @property({ type: Number }) 
  count = 0;

  // 3. 声明式渲染 (数据变了，这里自动跑)
  render() {
    return html`
      <p>计数: ${this.count}</p>
      <button @click=${() => this.count++}>自增</button>
    `;
  }
}
```
